# 人工智能训练师三级实践题整理规范

## 一、整理目标

帮助学生真正理解每道实践题的代码含义，避免死记硬背。只整理**空缺部分**和**需要理解的核心代码**，不重复考试原题和考试要求。

## 二、每道题文档位置

```
analysis/<数字目录>/README.md
```

例如：`analysis/1.1.1/README.md`

## 三、每道题文档结构

### 1. 题目大意

用 **1-2 句话** 概括这道题在做什么。不要复述完整考试要求。

### 2. 空缺部分答案 + 代码解释

这是文档的核心。按照原始代码中的空缺顺序，逐个给出答案，并解释这行/这段代码在做什么。

- 每个空单独成段。
- 给出完整填空后的代码。
- 解释要通俗易懂，学生能看懂。
- 对于非空缺的代码（如题目里已经写好的 `apply(lambda x: ...)`, `.mean()` 等），如果学生需要理解，也要解释，但不要把它当成"答案"。

### 3. 用到的 Python 函数（仅空缺部分）

只列出**空缺处真正出现**的 Python 函数/方法。题目里已经写好的函数（如 `apply`, `lambda`, `mean` 等）不列在这里。

格式：

```markdown
| 函数 | 含义 |
|------|------|
| `pd.read_csv()` | 读取 CSV 文件 |
```

### 4. 需要背的单词/术语（仅空缺部分）

只列出**空缺处真正出现**的英文单词、列名、参数名等。如果某道题确实没有需要背的单词，就**不写这一节**。

格式：

```markdown
| 单词 | 含义 |
|------|------|
| `read_csv` | 读取 CSV |
```

### 5. 注意事项

列出这道题的易错点、关键理解点、考试容易丢分的地方。控制在 5-10 条以内，要精简。

## 四、不整理的内容

- 不整理完整考试原题。
- 不整理完整考试要求。
- 不整理与空缺无关的函数和单词。
- 不添加代码示例（解释里必要的代码片段除外）。
- 不做过度扩展（如这道题没问的参数、用法等）。

## 五、汇总文档规范

等 40 道题全部整理完成后，在 `analysis/` 根目录下生成一个汇总文档：

- 汇总所有 40 道题的 Python 函数（去重）。
- 汇总所有 40 道题的单词/术语（去重，仅包含确实出现过的）。
- 汇总所有 40 道题的注意事项（去重/合并同类项）。
- 汇总文档中同样要给解释，但保持简洁。
